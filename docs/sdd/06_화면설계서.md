# 06. 화면설계서

## 화면 공통 원칙
- 조회 검색의 기본 입력은 `정규화 부품명`, `색상`이어야 한다.
- 화면에 노출하는 대표 날짜는 `포장일자`여야 한다.
- 위험도 표시는 `최초투입일자` 기준 계산 결과여야 한다.

## 조회 화면 초안

### 화면명: 생산계획/부족 조회
- 입력 항목
- `정규화 부품명`
- `색상`
- 위험도 필터
- 날짜 필터: 미정
- 결과 표시
- `정규화 부품명`
- `색상`
- `제품코드`
- `포장일자`
- 위험도
- 총 필요수량
- 현재 재공
- 부족수량
- 동일 부품의 여러 일정은 합산 1행으로 노출

### 화면명: 부품 상세
- 입력 항목
- `정규화 부품명`
- `색상`
- 결과 표시
- 부품 기본정보
- 연관 제품코드 목록
- 부품코드 목록
- 부품별 필요수량
- `최초투입일자`
- `포장일자`
- 동일 부품 기준 일정 목록은 내부적으로 합산값과 제품코드 목록을 함께 제공

## 관리자 화면 초안
- 업로드 시 제품/부품/날짜 컬럼을 모두 검증해야 한다.
- 업로드 시 `부품명` 정규화 결과를 미리보기로 확인할 수 있으면 좋다.
- 수동 등록 화면은 제품 정보, 부품 정보, 정규화 부품명을 분리 입력하도록 바뀌어야 하는지 사용자 확인 필요

## 미확정 화면 항목
- 재공 입력 화면에서 다중 일정 상세 팝업이 필요한지 미정
