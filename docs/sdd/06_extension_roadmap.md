# 06. 확장 포인트 및 리팩토링 방향

## 확장 포인트

- 입고예정, 외주반출, 안전재고 테이블 추가
- 관리자 비밀번호 방식에서 사용자/권한 테이블 방식으로 전환
- ERP/MES 엑셀 대신 API 연동
- 외주업체별 전용 공유 URL 또는 토큰 방식 도입
- 변경이력 상세 diff 저장

## 리팩토링 방향

- Streamlit UI를 페이지 모듈 단위로 추가 분리
- 서비스 계층을 CRUD와 리포트 계층으로 분리
- 계산 로직을 테스트 가능한 순수 함수로 분리
- SQLite에서 Postgres로 이동할 수 있도록 SQL 의존성을 repository 패턴으로 캡슐화
- 업로드 스키마 검증을 별도 validator 모듈로 분리
