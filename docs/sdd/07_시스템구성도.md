# 07. 시스템구성도

## 현재 구성
- 프론트엔드: Streamlit
- 애플리케이션 코드: Python
- 데이터 저장소: Supabase
- 입력 소스: Excel 파일 업로드

## 현재 처리 계층
1. `app.py`
2. `urethane_wip/data_loader.py`
3. `urethane_wip/services.py`
4. `urethane_wip/db.py`
5. Supabase 테이블

## 구조 이슈
- 로더는 제품과 부품을 함께 읽지만 서비스 계층에서 부품코드 기준 키로 수렴한다.
- 제품코드와 부품코드의 관계를 전용 구조로 보존하지 않는다.
- 정규화 부품명 계층이 없다.

## 목표 구성 변화
- 데이터 계층에서 제품 정보, 부품 정보, 정규화 부품명을 분리
- 제품코드와 부품코드의 매핑 테이블을 추가
- 서비스 계층에서 관리 기준 키를 `정규화 부품명 + 색상`으로 통일
- 화면 계층에서 캘린더에 제품코드를 보조 표시
