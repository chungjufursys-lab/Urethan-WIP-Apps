# 08. 인터페이스정의서

## 입력 인터페이스

### Excel 업로드
- 파일: `가공실적등록.xls`
- 성격: 부품단위 계획 파일
- 확인된 입력 컬럼
- `부품이동카드번호`
- `최초투입일자`
- `포장일자`
- `제품코드`
- `제품색상`
- `제품명`
- `부품코드`
- `부품색상`
- `부품명`
- `계획량`
- `생산수량`
- `건명`

## 내부 인터페이스

### 현재 data_loader -> production_plan
- 제품: `product_code`, `product_name`
- 부품: `urethane_item_code`, `color`
- 날짜: `plan_date`, `due_date`

### 목표 data_loader -> production_plan
- 제품: `product_code`, `product_color`, `product_name`
- 부품: `part_color`, `part_name_raw`, `part_name_normalized`
- 날짜: `first_input_date`, `package_date`
- 수량: `plan_qty`, `produced_qty`, `required_qty`

### 목표 data_loader -> code_mapping
- `product_code`
- `product_color`
- `part_color`
- `part_name_normalized`

### services -> 화면
- 현재 반환 키는 사실상 `부품코드 + 부품색상`
- 목표 반환 키는 `정규화 부품명 + 색상`
- 캘린더는 `제품코드`를 함께 반환해야 한다.
- 업체 매핑 화면도 `정규화 부품명 + 색상` 기준으로 반환해야 한다.
- 동일 부품의 여러 일정은 합산 1행으로 반환해야 한다.
