# 10. 테스트케이스

## 구조 진단 테스트
- `TC-001`
- 전제 조건: `가공실적등록.xls` 업로드 가능
- 입력: 제품코드, 제품색상, 부품코드, 부품색상이 모두 존재하는 행
- 예상 결과: 제품 컬럼과 부품 컬럼이 각각 독립 저장된다.
- 관련 요구사항: `FR-002`, `FR-007`

- `TC-002`
- 전제 조건: 생산계획 데이터 존재
- 입력: 특정 `정규화 부품명 + 색상`
- 예상 결과: 조회는 부품명 기준으로 검색되고, 상세에서 연관 제품코드가 노출된다.
- 관련 요구사항: `FR-001`, `FR-004`, `FR-008`

- `TC-003`
- 전제 조건: 날짜 컬럼 존재
- 입력: `최초투입일자`, `포장일자`
- 예상 결과: 위험도는 최초투입일자 기준, 화면 날짜는 포장일자 기준으로 동작한다.
- 관련 요구사항: `FR-005`, `FR-006`

- `TC-004`
- 전제 조건: 업로드 원본에 `AAA[C20~]` 형태의 부품명이 존재
- 입력: 원본 부품명
- 예상 결과: 저장 시 원본 부품명과 정규화 부품명이 모두 확인 가능하고, 조회는 정규화 부품명 기준으로 동작한다.
- 관련 요구사항: `FR-003`

- `TC-005`
- 전제 조건: 제품코드와 정규화 부품명이 N:1 관계인 데이터 존재
- 입력: 계획 데이터 적재
- 예상 결과: 매핑 테이블에서 제품코드와 정규화 부품명 관계를 추적할 수 있다.
- 관련 요구사항: `FR-004`

- `TC-006`
- 전제 조건: 업체 매핑 데이터 존재
- 입력: 업체 매핑 조회
- 예상 결과: 화면 노출 기준은 `정규화 부품명 + 색상`이다.
- 관련 요구사항: `FR-001`, `FR-008`

- `TC-007`
- 전제 조건: 동일 `정규화 부품명 + 색상`에 여러 일정 건이 존재
- 입력: 생산계획/부족 조회 화면
- 예상 결과: 화면에는 합산 1행으로 노출된다.
- 관련 요구사항: `FR-001`

## 미정 테스트
