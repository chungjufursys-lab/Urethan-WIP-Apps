# 03. 기능 요구사항

## 기능 요구사항

### 공개 조회

- `FR-001` 로그인 없이 전체 재공 현황을 조회할 수 있어야 한다.
- `FR-002` 품목, 색상, 위치, 상태 기준 필터를 제공해야 한다.
- `FR-003` 현재 재공 수량과 계획 필요량, 부족분을 함께 표시해야 한다.
- `FR-004` 외주업체가 참고할 수 있는 일정/부족 현황을 별도 화면으로 제공해야 한다.
- `FR-005` 조회 결과를 CSV로 다운로드할 수 있어야 한다.

### 관리자

- `FR-006` 관리자 화면은 비밀번호 인증 후에만 접근 가능해야 한다.
- `FR-007` 재공 데이터를 추가/수정/삭제할 수 있어야 한다.
- `FR-008` 품목 마스터와 위치 마스터를 관리할 수 있어야 한다.
- `FR-009` 생산계획을 수기 입력하거나 엑셀에서 업로드할 수 있어야 한다.
- `FR-010` 외주 공유 데이터를 등록/수정/삭제할 수 있어야 한다.
- `FR-011` 변경 시각과 수정자명을 저장해야 한다.

### 계산

- `FR-012` 부족분 = 계획 필요량 - 현재 재공 공식을 기본으로 사용해야 한다.
- `FR-013` 부족분이 양수이고 납기가 임박한 항목은 위험 상태로 표시해야 한다.
- `FR-014` 구조상 입고예정, 외주반출, 안전재고 컬럼을 나중에 추가하기 쉬워야 한다.

## 비기능 요구사항

- `NFR-001` SQLite 단일 파일로 바로 실행 가능해야 한다.
- `NFR-002` 코드는 파일 분리와 주석을 포함해 유지보수 가능해야 한다.
- `NFR-003` 운영자가 빠르게 배울 수 있는 단순한 화면 구성이어야 한다.

## 가정

- 사용자가 언급한 `비고=외주입고` 조건은 현재 샘플 파일에서는 `건명=외주입고분`으로 대응한다.
- 관리자 인증은 단일 공유 비밀번호 방식으로 시작한다.
