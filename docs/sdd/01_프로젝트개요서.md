# 01. 프로젝트개요서

## 프로젝트명
- 우레탄 WIP Manager 구조 재정의

## 배경
- 현재 애플리케이션은 Streamlit 화면, Supabase 저장소, Excel 업로드를 사용한다.
- 현재 문제의 핵심은 제품코드와 부품코드가 N:1로 연결되는데, 이 매칭이 구조적으로 정리되어 있지 않다는 점이다.
- 현재 구현은 주로 부품코드 기준으로 조회/집계되지만, 부품명 정규화와 코드 매핑 구조가 없다.

## 목적
- 부품단위 계획 파일 기준으로 데이터 구조를 다시 정의한다.
- 제품코드와 부품코드의 연결 관계를 별도 매핑 구조로 분리한다.
- 부품명 표기 흔들림을 정규화해 관리 기준을 안정화한다.

## 현재 확정 정보
- 업로드 파일은 부품단위 계획 파일이다.
- 별도 BOM 테이블은 현재 필요하지 않다.
- `부품명`에서 `[C20~]` 같은 문자열을 제거하고 인식해야 한다.
- 구조 분리 후의 관리 기준은 `부품명 정규화값 + 색상`이다.
- 캘린더 노출에는 보조 정보로 `제품코드`를 함께 표시한다.
- 위험도 계산 기준 날짜는 `최초투입일자`다.
- 화면 노출 기준 날짜는 `포장일자`다.

## 범위
- 현재 DB/코드 구조 진단
- 목표 데이터 구조 정의
- 코드 매핑 테이블 설계
- 부품명 정규화 규칙 정의
- 요구사항, 업무규칙, 예외정책, 테스트 기준 정리

## 제외 범위
- 별도 BOM 설계
- ERP/MDB 직접 연동 상세
- 실제 마이그레이션 SQL 작성

## 기대 효과
- 제품코드와 부품코드의 N:1 관계를 명시적으로 관리할 수 있다.
- 부품명 표기 흔들림을 흡수해 동일 부품 인식을 안정화할 수 있다.
- 위험도 계산 기준과 화면 노출 기준 날짜를 분리해 관리할 수 있다.
