# 09. 예외처리정책서

## 확인된 예외 유형
- `EX-001` 업로드 필수 컬럼 누락
- `EX-002` 제품 기준 조회 KEY와 저장 기준 KEY 불일치
- `EX-003` 업체 매핑 조인 실패
- `EX-004` 날짜 컬럼 의미 혼동
- `EX-005` 동일 부품명에 `[C20~]` 같은 문자열 포함으로 인한 중복 인식

## 현재 확인된 예외 현상
- 현재 구조에서는 생산계획과 업체 매핑의 기준 키가 달라 `vendor_name`이 비어 있는 행이 발생한다.
- 현재 구조에서는 정규화 부품명 관리가 없어 부품명 표기 흔들림을 흡수하지 못한다.

## 예외 처리 원칙 초안
- `EXR-001` 필수 컬럼이 없으면 업로드를 중단하고 누락 컬럼을 보여준다.
- `EXR-002` 제품 정보와 부품 정보는 서로 대체하지 않는다.
- `EXR-003` 미정 규칙은 기본값으로 대체하지 않고 확인 필요 상태로 남긴다.
- `EXR-004` 부품명 정규화 전후 값을 모두 추적 가능해야 한다.
- `EXR-005` 대괄호 제거 후 빈 문자열이 되는 부품명은 오류로 처리할지 사용자 확인 필요하다.

## 미정
- 업로드 건 일부만 오류일 때 전체 중단인지 부분 반영인지 사용자 확인 필요
- 동일 KEY 중복 처리 규칙 사용자 확인 필요
