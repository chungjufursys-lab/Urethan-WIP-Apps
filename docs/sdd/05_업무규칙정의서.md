# 05. 업무규칙정의서

## 확정 업무규칙
- `BR-001` 모든 조회와 관리의 기본 기준은 `정규화 부품명 + 색상`이다.
- `BR-002` 업로드 파일은 부품단위 계획 파일로 간주한다.
- `BR-003` 별도 BOM 테이블은 두지 않는다.
- `BR-004` 기존 부품코드의 역할은 정규화 부품명으로 대체한다.
- `BR-005` 제품코드와 정규화 부품명의 N:1 관계는 매핑 테이블로 관리한다.
- `BR-005` `부품명`은 파싱 시 `[C20~]` 같은 문자열을 제거해 정규화한다.
- `BR-006` 위험도 계산 기준 날짜는 `최초투입일자`다.
- `BR-007` 화면 노출 기준 날짜는 `포장일자`다.
- `BR-008` 캘린더에는 기준 부품 외에 `제품코드`를 함께 노출한다.
- `BR-009` 업체 매핑 노출 기준은 `정규화 부품명 + 색상`이다.
- `BR-010` 모든 노출 테이블은 `정규화 부품명 + 색상` 기준으로 관리한다.
- `BR-011` 대괄호로 둘러싸인 문자열은 부품명 정규화 시 모두 제거한다.
- `BR-012` 동일 부품에 여러 일정이 있으면 화면에서는 합산 1행으로 보여준다.

## 현재 구조와 충돌하는 규칙
- `BR-013` 현재 구현은 `부품코드 + 부품색상`을 조회 KEY처럼 사용하고 있으므로 목표 규칙과 충돌한다.
- `BR-014` 현재 구현은 정규화 부품명 관리가 없어 동일 부품의 표기 흔들림을 흡수하지 못한다.
- `BR-015` 현재 구현은 제품코드와 정규화 부품명의 연결 관계를 전용 구조로 관리하지 않는다.
- `BR-016` 현재 구현은 다중 일정 건을 화면 합산 1행 기준으로 정리하지 않는다.

## 미확정 업무규칙
- `BR-017` 동일 부품에 여러 포장일자가 있을 때 화면 대표값 선정 규칙은 미정
- `BR-018` 완료 상태에서 `required_qty`를 0으로 만들지 여부는 사용자 확인 필요
